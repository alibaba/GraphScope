使用 NeuG 探索社交网络

欢迎来到这个使用 TinySNB（Tiny Social Network Benchmark）数据集的综合教程！本 Python 教程将引导你探索一个小型社交网络图数据库，展示使用 NeuG 进行社交网络分析的图查询的强大功能。

什么是 TinySNB？

TinySNB 由 Kuzu  提供用于测试目的，是一个小型社交网络数据集，模拟了人、组织和电影之间的关系。它是学习图数据库概念和针对合成的真实用例测试查询的理想选择。

该数据集包含：

  • 人物：具有个人信息的个体（姓名、年龄、职业状况等）
  • 组织：人们学习或工作的大学和公司
  • 电影：带有评分和描述的影片
  • 关系：社交连接、工作关系、学术关联等等

快速开始

让我们首先加载 TinySNB 数据集并探索其结构。 如果你还没有安装，请先安装 NeuG。

加载数据集

import neug
import os
 
db_path = '/path/to/database'
 
if not os.path.exists(db_path):
    # 首先，将内置的 TinySNB 数据集加载到新数据库中
    db = neug.Database(db_path)
    db.load_builtin_dataset('tinysnb')
else:
    # 如果路径存在，则直接打开数据库而无需额外加载
    db = neug.Database(db_path)
 
conn = db.connect()
 
print("TinySNB 数据集加载成功！")

### 基础人物查询

让我们从探索我们社交网络中的人物开始：

```python
# 获取所有人的基本信息
print("=== 我们社交网络中的人 ===")
result = conn.execute("""
    MATCH (p:person) 
    RETURN p.fName, p.age, p.isStudent, p.isWorker 
    ORDER BY p.age
""")
 
for record in result:
    name, age, is_student, is_worker = record
    status = []
    if is_student:
        status.append("Student")
    if is_worker:
        status.append("Worker")
    status_str = " & ".join(status) if status else "Neither student nor worker"
    print(f"{name} (年龄 {age}): {status_str}")

## 社交网络分析：关系

现在让我们探索人与人之间的关系——这正是图数据库真正擅长的地方！

### 谁认识谁？

```python

# 探索 "knows" 关系
print("=== 社交关系（谁认识谁）===")
result = conn.execute("""
    MATCH (p1:person)-[k:knows]->(p2:person)
    RETURN p1.fName, p2.fName, k.date
    ORDER BY p1.fName, p2.fName
""")

for record in result:
    print(f"{record[0]} 认识 {record[1]} (自 {record[2]})")

寻找受欢迎的人

 
# 谁有最多的关系？
print("\n=== 最多连接的人 ===")
result = conn.execute("""
    MATCH (p:person)-[k:knows]->(friend:person)
    RETURN p.fName, count(friend) as friend_count
    ORDER BY friend_count DESC
    LIMIT 5
""")
 
for record in result:
    print(f"{record[0]} 认识 {record[1]} 个人")
 
# 谁被最多人认识？
print("\n=== 最受欢迎的人（被他人认识）===")
result = conn.execute("""
    MATCH (p:person)<-[k:knows]-(friend:person)
    RETURN p.fName, count(friend) as known_by_count
    ORDER BY known_by_count DESC
    LIMIT 5
""")
 
for record in result:
    print(f"{record[0]} 被 {record[1]} 个人认识")

相互连接

print("\n=== 相互友谊 ===")
result = conn.execute("""
    MATCH (p1:person)-[k1:knows]->(p2:person),
          (p2:person)-[k2:knows]->(p1:person)
    WHERE p1.id < p2.id  // 避免重复
    RETURN p1.fName, p2.fName
    ORDER BY p1.fName
""")
 
for record in result:
    print(f"{record[0]} 和 {record[1]} 相互认识")

高级模式匹配

多跳关系

 
# 查找朋友的朋友（2度连接）
print("=== 朋友的朋友（2度连接）===")
result = conn.execute("""
    MATCH (p1:person)-[:knows]->(mutual:person)-[:knows]->(p2:person)
    WHERE p1.id <> p2.id  // 不同的人
    AND NOT (p1)-[:knows]-(p2)  // 不是直接朋友
    RETURN p1.fName, p2.fName, mutual.fName
    ORDER BY p1.fName
""")
 
for record in result:
    print(f"{record[0]} 可以通过 {record[2]} 认识 {record[1]}")

```python

# 查找连接最紧密的个体（网络中心）
print("\n=== 网络中心（连接最紧密的个体）===")
result = conn.execute("""
    MATCH (p:person)
    OPTIONAL MATCH (p)-[out:knows]->()
    OPTIONAL MATCH (p)<-[i:knows]-()
    RETURN p.fName, 
           count(DISTINCT out) as outgoing,
           count(DISTINCT i) as incoming,
           count(DISTINCT out) + count(DISTINCT i) as total_connections
    ORDER BY total_connections DESC
    LIMIT 5
""")

for record in result:
    print(f"{record[0]}: {record[3]} 总连接数（{record[1]} 个出向连接，{record[2]} 个入向连接）")

基于年龄的社交分析

 
# 按年龄组分析社交关系
print("\n=== 各年龄组的社交关系 ===")
result = conn.execute("""
    MATCH (p1:person)-[:knows]->(p2:person)
    WITH p1, p2,
         CASE 
             WHEN p1.age < 25 THEN "年轻 (< 25)"
             WHEN p1.age < 35 THEN "成年 (25-34)" 
             WHEN p1.age < 50 THEN "中年 (35-49)"
             ELSE "老年 (50+)"
         END as age_group1,
         CASE 
             WHEN p2.age < 25 THEN "年轻 (< 25)"
             WHEN p2.age < 35 THEN "成年 (25-34)"
             WHEN p2.age < 50 THEN "中年 (35-49)" 
             ELSE "老年 (50+)"
         END as age_group2
    RETURN age_group1, age_group2, count(*) as connection_count
    ORDER BY connection_count DESC
""")
 
for record in result:
    print(f"{record[0]} → {record[1]}: {record[2]} connections")
 
## 总结
 
在本教程中，你已经学会了如何：
 
1. 使用 NeuG 的数据集功能**加载内置数据集**
2. **探索图模式**并理解数据结构
3. 执行**基本查询**来查找和过滤节点
4. **分析图中实体之间的关系**
5. 使用**模式匹配**来查找复杂关系和路径
6. 计算**网络指标**以了解社交网络属性
7. **组合多种关系类型**从互连数据中获得洞察
 
### 关键要点
 
- **图数据库擅长关系查询**：查找像"朋友的朋友"或"既是同事又是朋友"这样的模式是自然且高效的
- **模式匹配功能强大**：在 SQL 中需要多次连接的复杂查询，在图数据库中变成直观的图模式
- **社交网络分析**：图数据库为分析网络结构、连通性和影响力提供了内置支持
