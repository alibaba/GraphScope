NeugDB

全名： neug::NeugDB

NeuG 图数据库系统的内核数据库引擎。

NeugDB 作为所有 NeuG 图数据库操作的主要入口点。它提供完整的生命周期管理 API，包括数据库初始化、查询执行和优雅关闭。

使用示例：

// 创建并打开数据库
neug::NeugDB db;
db.Open("/path/to/data", 4);  // 4 个线程
// 创建连接并执行查询
auto conn = db.Connect();
auto result = conn->Query("MATCH (n:Person) RETURN n LIMIT 10");
// 处理结果
for (auto& record : result.value()) {
  std::cout << record.ToString() << std::endl;
}
// 关闭数据库（持久化数据）
db.Close();

核心组件：

  • PropertyGraph：底层图数据存储引擎
  • QueryProcessor：Cypher 查询编译和执行
  • ConnectionManager：客户端连接池管理
  • IGraphPlanner：查询优化（GOPT 或贪心规划器）

数据库模式：

  • DBMode::READ_ONLY：用于分析工作负载的只读访问
  • DBMode::READ_WRITE：完全事务性读写访问

线程安全： 此类是线程安全的。多个连接可以并发执行查询。ConnectionManager 在内部处理线程同步。

资源管理：

  • 文件锁定防止多个进程并发访问数据库
  • 自动 WAL（预写日志）用于崩溃恢复
  • 可配置的检查点和关闭时压缩

公共方法

Open(...)

Open(
    const std::string &data_dir,
    int32_t max_num_threads=0,
    const DBMode mode=DBMode::READ_WRITE,
    const std::string &planner_kind="gopt",
    bool enable_auto_compaction=false,
    bool compact_csr=true,
    bool compact_on_close=true,
    bool checkpoint_on_close=true
)

从持久化存储中打开数据库。

从指定的数据目录初始化并打开 NeuG 数据库。此方法加载图模式、顶点/边数据，并初始化查询处理器和规划器。

数据目录结构： data_dir 应包含：

  • graph.yaml：Schema 定义文件
  • snapshot/：顶点和边数据文件
  • wal/：预写日志文件（可选，用于恢复）

使用示例：

neug::NeugDB db;
// 使用默认值简单打开
db.Open("/path/to/graph");
// 使用自定义设置打开（8个线程，读写模式，GOPT 规划器）
db.Open("/path/to/graph", 8, neug::DBMode::READ_WRITE, "gopt");

  • 参数：

    • data_dir：图数据目录的路径
    • max_num_threads：并发操作的最大线程数。如果为 0，则使用硬件并发性（CPU 核心数）
    • mode：数据库访问模式（READ_ONLY 或 READ_WRITE）
    • planner_kind：查询规划器类型：“gopt”（图优化器）或 “greedy”
    • enable_auto_compaction：启用后台自动压缩线程
    • compact_csr：在自动压缩期间压缩 CSR 结构
    • compact_on_close：关闭数据库时执行压缩
    • checkpoint_on_close：关闭时创建检查点（持久化数据）

  • 注意：

    • 此重载主要为 Python 绑定设计。
    • 对于 C++ 使用，请优先选择基于配置的 Open(NeugDBConfig&) 重载。

  • 返回： 如果数据库成功打开则返回 true，否则返回 false

  • 自从： v0.1.0

Open(const NeugDBConfig &config)

使用配置对象打开数据库。

使用提供全面配置选项的 NeugDBConfig 结构打开数据库。

使用示例：

neug::NeugDBConfig config;
config.data_dir = "/path/to/graph";
config.thread_num = 8;
config.mode = neug::DBMode::READ_WRITE;
config.memory_level = 2;  // 使用 hugepages
config.enable_auto_compaction = true;
neug::NeugDB db;
db.Open(config);

  • 参数：

    • config：包含所有数据库设置的配置对象

  • 返回值： 如果数据库成功打开则返回 true，否则返回 false

  • 自版本： v0.1.0

Close()

关闭数据库并释放所有资源。

执行数据库的优雅关闭。根据配置：

  • 如果启用了 checkpoint_on_close，则创建检查点
  • 如果启用了 compact_on_close，则执行压缩
  • 关闭所有打开的连接
  • 释放文件锁

重要： 在销毁 NeugDB 实例之前，始终调用 Close() 以确保数据完整性和正确的资源清理。

使用示例：

neug::NeugDB db;
db.Open("/path/to/data");
// ... 执行操作 ...
db.Close();  // 持久化数据并清理

  • 注意事项：

    • 此方法是幂等的 - 多次调用是安全的。
    • 关闭后，数据库无法重新打开。要再次打开数据库，请创建一个新的 NeugDB 实例。

  • 自： v0.1.0

IsClosed() const

检查数据库是否已关闭。

  • 返回值： 如果数据库已关闭则返回 true。

Connect()

创建一个到数据库的新连接以执行查询。

创建并返回一个 Connection 对象，该对象可用于对数据库执行 Cypher 查询。此连接与来自同一数据库的其他连接共享查询规划器和处理器。

使用示例：

auto conn = db.Connect();
auto result = conn->Query("MATCH (n) RETURN count(n)");
if (result.has_value()) {
    std::cout << "Query succeeded" << std::endl;
}
conn->Close();  // 可选：析构时自动关闭

  • 注意事项：

    • 在 READ_ONLY 模式下，可以创建多个连接。
    • 在 READ_WRITE 模式下，只允许一个写连接。
    • 连接共享规划器实例以提高效率。

  • 抛出异常：

    • std::runtime_error：如果数据库未打开或已关闭

  • 返回值： std::shared_ptr 指向新 Connection 的共享指针

  • 自版本： v0.1.0

RemoveConnection(std::shared_ptr< Connection > conn)

从数据库中移除一个连接。

  • 参数：

    • conn：要移除的连接。

  • 注意事项：

    • 此方法用于在连接关闭时将其从数据库中移除句柄。
    • 此方法不是线程安全的，因此只应在连接关闭时调用。并且应该仅在内部调用。
