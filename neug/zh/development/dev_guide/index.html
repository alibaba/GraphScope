开发者指南

从源码构建

要从源码编译 NeuG，必须安装某些依赖项和工具。

由于几乎所有依赖项也作为第三方库包含在 NeuG 仓库中，你只需安装少数几个必要的包即可在本地构建 NeuG。

在 Ubuntu 上

sudo apt-get update
sudo apt-get install -y build-essential cmake git python3-dev python3-pip g++ make libssl-dev openssl

在 macOS 上

brew update
brew install cmake git python3 openssl@3
 
# 无需安装 g++，因为也支持 Apple Clang。

在 CentOS7 上

更新 yum 仓库以在 CentOS 7 上使用 vault.centos.org，因为主镜像源已不再可用。

sed -i “s/mirror.centos.org/vault.centos.org/g” /etc/yum.repos.d/.repo &&
sed -i “s/^#.baseurl=http/baseurl=http/g” /etc/yum.repos.d/.repo &&
sed -i “s/^mirrorlist=http/#mirrorlist=http/g” /etc/yum.repos.d/.repo sudo yum -y install centos-release-scl sed -i “s/mirror.centos.org/vault.centos.org/g” /etc/yum.repos.d/.repo &&
sed -i “s/^#.baseurl=http/baseurl=http/g” /etc/yum.repos.d/.repo &&
sed -i “s/^mirrorlist=http/#mirrorlist=http/g” /etc/yum.repos.d/.repo sudo yum -y install epel-release sudo yum -y groupinstall “Development Tools” sudo yum -y install git python3 python3-pip make cmake3 openssl openssl-devel sudo ln -sf /usr/bin/cmake3 /usr/local/bin/cmake

通过 devtoolset-10 安装较新版本的 gcc/g++

sudo yum -y install devtoolset-10 scl enable devtoolset-10 bash


#### 在 CentOS8/CentOS Stream 8 上

```bash
sudo dnf -y install epel-release dnf-plugins-core

# 启用 PowerTools（CentOS 8 中称为 PowerTools；Stream 8 中通常为 powertools）
sudo dnf config-manager --set-enabled powertools || sudo dnf config-manager --set-enabled PowerTools
sudo dnf -y groupinstall "Development Tools"
sudo dnf -y install git python3 python3-pip cmake gcc-c++ make

构建 NeuG

环境准备就绪后，你可以继续构建 NeuG。

用于开发目的

要在开发模式下构建 NeuG Python 包，请执行：

make python-dev

这将在开发模式下编译 NeuG 引擎和 Python 客户端。你可以通过从 tools/python_bind 目录运行 Python 脚本来使用 NeuG 包。

cd tools/python_bind
python3
>>> import neug

要从其他目录使用 NeuG，请将 tools/python_bind 添加到 sys.path：

import sys
sys.path.append('/path/to/neug/tools/python_bind')

构建 Wheel 包

要编译 wheel 包，请运行：

make python-wheel

之后，wheel 包可以在 tools/python_bind/dist 中找到。

仅构建 C++ 库和可执行文件

要仅构建 C++ 库和可执行文件而不构建 Python 绑定，请使用：

mkdir build && cd build
cmake ..
make -j$(nproc)
 
# 可选：运行测试
ctest
 
# 可选：安装到系统
make install

查看 CMakeLists.txt 获取更多 CMake 选项。

构建选项

你可以通过指定以下环境变量来设置不同的选项，从而自定义构建过程：

export BUILD_EXECUTABLES=ON/OFF # 切换是否构建实用程序可执行文件
export BUILD_HTTP_SERVER=ON/OFF # 在 NeuG 中启用或禁用 HTTP 服务器支持
export WITH_MIMALLOC=ON/OFF # 决定是否使用 mimalloc 替代 glibc 的默认 malloc
export ENABLE_BACKTRACES=ON/OFF # 使用 cpptrace 链接 NeuG 库以在异常时提供详细的堆栈跟踪
export BUILD_TYPE=DEBUG/RELEASE # 设置 CMake 构建类型
export BUILD_TEST=ON/OFF # 切换是否构建测试套件

调试

默认情况下，C++ 日志记录是禁用的。要启用日志记录，请使用：

export DEBUG=ON

如需更详细的日志记录，可以通过以下命令调整 glog 详细程度级别：

export GLOG_v=10 # 全局设置
GLOG_v=10 python3 ... # 为单个命令设置

要进一步调查段错误或其他复杂问题，建议使用 gdb/lldb：

GLOG_v=10 gdb --args python3 -m pytest -sv tests/test_db_query.py
GLOG_v=10 lldb -- python3 -m pytest -sv tests/test_db_query.py

有关其他调试技术，请分别参考 gdb 和 lldb 的文档。

本地预提交检查

在将代码推送到 GitHub 之前，运行本地检查以尽早发现问题并节省 CI 资源：

提示: 检查需要 Python 环境。 使用 python3 -m venv .venv && source .venv/bin/activate 或 conda create -n neug python=3.13 && conda activate neug 来设置环境。

 
# 仅格式检查（快速，建议提交前运行）
make format-check
 
# 完整检查，包括构建和测试（建议创建 PR 前运行）
make full-check

format-check 验证 C++（clang-format）和 Python（isort、black、flake8）代码格式。任何可自动修复的问题（clang-format、isort、black）将就地修正；只有 flake8 问题需要手动修复。 full-check 还会编译代码并运行单元测试。

有关更多选项，请参见 ./scripts/pre_commit_check.sh --help。
