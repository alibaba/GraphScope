开发指南AI 辅助开发

AI 辅助开发

NeuG 提供 AI 辅助技能来简化你的开发工作流程。这些技能与你的编码代理（Cursor、Qoder、Qwen Code 等）集成作为代理技能。你可以使用斜杠命令来调用它们。

快速开始

1. 为你的编码代理初始化技能：

./scripts/init_skills.sh <agent>   # 例如，qoder, qwen

注意： Cursor 是默认代理，无需初始化。

查看 支持的代理 获取完整列表。

2. 在聊天框中使用以 / 前缀的技能：

/create-issue 我在运行这个查询时遇到了段错误...

常用技能

/create-issue - 报告错误或请求功能

直接从你的 IDE 创建 GitHub issue，并提供完整上下文。

示例：报告 Bug

/create-issue
Type: Bug
Assignee: @who
Parent Issue: #42

查询导致段错误。终端输出：

E20260112 10:23:45.123456 12345 executor.cc:271] Query execution failed
Segmentation fault (core dumped)

示例：请求功能

/create-issue
Type: Feature
Assignee: @me

我需要支持带有多个模式的 OPTIONAL MATCH。
目前只支持单个模式。

提示：

  • 直接复制粘贴终端错误输出 - 代理会提取关键信息
  • 指定 Type: Bug 或 Type: Feature 来跳过自动检测
  • 使用 Assignee: @me 或 Assignee: @username 进行预分配
  • 使用 Parent Issue: #id 链接为子问题

/create-pr - 提交拉取请求

创建带有自动生成摘要的 PR，链接到相关问题。

示例

/create-pr
Fixes: #42
Reviewers: @who

这修复了查询执行中的段错误问题。
在访问结果缓冲区之前添加了空指针检查。

提示：

  • 使用 Fixes: #id 在 PR 合并时自动关闭问题
  • 使用 Reviewers: @user1, @user2 请求特定审阅者
  • 代理会根据你的更改自动生成 PR 标题和摘要

/update-with-comments - 处理 PR 反馈

自动获取并应用 PR 审阅评论。

示例

/update-with-comments

或指定一个 PR：

/update-with-comments #123

代理将：

  1. 获取 PR 中的所有审阅评论
  2. 将请求的更改应用到你的代码中
  3. 显示修改摘要
  4. 在你审阅后提交并推送

规范驱动的工作流程

对于较大的功能，我们提供了一个结构化的工作流程，在实现之前进行规划。

步骤 1：/speckit.specify - 定义功能

从自然语言创建功能规格说明。

示例

/speckit.specify
为 NeuG 添加图算法扩展支持。
用户应该能够注册自定义算法并通过 Cypher 执行它们。

代理将：

  1. 创建一个功能分支（例如，001-graph-algo-extension）
  2. 在 specs/001-graph-algo-extension/spec.md 中生成规格文档
  3. 创建一个 GitHub issue 来跟踪该功能
  4. 如需要，提出澄清问题（最多 3 个）

步骤 2：/speckit.plan - 计划实现

根据规格说明生成技术计划。

示例

/speckit.plan 001-graph-algo-extension

代理将：

  1. 分析规格说明和代码库结构
  2. 规划项目结构、数据模型和算法
  3. 生成包含实现细节的 plan.md
  4. 创建一个关联的 GitHub issue

步骤 3：/speckit.tasks - 分解为任务

从计划中生成可执行的任务。

示例

/speckit.tasks 001-graph-algo-extension

代理将：

  1. 创建模块级分解
  2. 为每个模块生成任务文件
  3. 每个任务的规模适合独立测试

同步到 GitHub

创建规格/计划/任务后，将它们同步到 GitHub：

/sync-modules 001-graph-algo-extension    # 同步模块问题
/sync-tasks 001-graph-algo-extension      # 同步任务问题

设置

先决条件

代理技能通过 markdown 文件实现，可以由编码代理自动调用或用户显式调用。

为了避免意外调用，我们默认为所有技能设置 disable-model-invocation: true。你只能通过斜杠命令调用它们。你也可以自定义技能文件以启用模型调用。

安装技能

我们在 .cursor/skills/ 目录下以 Cursor 格式提供代理技能。要为你的代理进行转换：

./scripts/init_skills.sh <agent-shortcut>
